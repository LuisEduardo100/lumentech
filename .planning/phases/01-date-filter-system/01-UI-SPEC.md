## Component Inventory

### DateFilterBar (new component: `frontend/src/components/DateFilterBar.tsx`)

A pill button group rendered in the header bar immediately after the category tabs group (D-02). Only visible when `view === 'dashboard'` (D-02 + existing App.tsx line 183 pattern).

**Container:**
```
flex items-center p-1 rounded-lg border transition-colors
light: bg-slate-100 border-slate-200
dark:  bg-slate-800 border-slate-700
```

**Inactive pill:**
```
px-3 py-1.5 rounded-md text-xs sm:text-sm font-bold transition-all
light: text-slate-500 hover:text-slate-700
dark:  text-slate-400 hover:text-slate-200
```

**Active pill (any category — uses neutral slate, not orange):**
```
px-3 py-1.5 rounded-md text-xs sm:text-sm font-bold transition-all
bg-slate-600 text-white shadow-md
```

Source: App.tsx lines 184-204 — exact replication of category tab classes

**Pill labels:**
- `SEMANA` (abbreviated from "Esta semana" to prevent header overflow — see Pitfall 4 in RESEARCH.md)
- `MÊS` (abbreviated from "Mês atual")
- `ANTERIOR` (abbreviated from "Mês passado")
- `PERSONALIZADO` (or dynamic date range when active, e.g. `01/03 - 31/03`)

Dynamic label when custom range is active: `format(customStart, 'dd/MM') + ' - ' + format(customEnd, 'dd/MM')` using date-fns `format`. Source: CONTEXT.md D-07; RESEARCH.md Pattern 5.

**Calendar icon:** Render `CalendarDays` (or `Calendar`) from lucide-react at `w-4 h-4` only inside the `PERSONALIZADO` pill, immediately before the label text. Inactive: no icon on preset pills.

### CustomRangePicker (inline popover within DateFilterBar)

Triggered by clicking the `PERSONALIZADO` pill (D-03). Rendered as an absolutely positioned `<div>` dropping below the pill.

**Popover container:**
```
absolute top-full mt-2 left-0 z-50
rounded-xl border shadow-xl p-4
flex flex-col gap-3
min-w-[240px]
light: bg-white border-slate-200
dark:  bg-slate-900 border-slate-700
```

`z-50` required to render above ECharts canvas elements (see RESEARCH.md Pitfall 5).

**Date inputs:** Two native `<input type="date">` fields, each with a text label above:

Label classes: `text-xs font-bold uppercase tracking-wide`
- light: `text-slate-500`
- dark: `text-slate-400`

Input classes:
```
px-2 py-1 rounded border text-sm w-full
light: bg-white border-slate-300 text-slate-800
dark:  bg-slate-700 border-slate-600 text-white
```

**Input labels (copy):**
- First input: `INÍCIO`
- Second input: `FIM`

**Validation error (inverted dates):** When `customStart > customEnd`, render a single line below the inputs:
```
text-xs text-red-500 font-semibold
```
Copy: `"Data final deve ser após a data inicial."`

Do not enable the "Aplicar" button when dates are inverted.

**Aplicar button:**
```
w-full px-4 py-2 rounded-lg text-sm font-bold transition-all
enabled:  bg-slate-700 text-white hover:bg-slate-600
disabled: bg-slate-200 text-slate-400 cursor-not-allowed (light)
          bg-slate-800 text-slate-600 cursor-not-allowed (dark)
```

**Dismiss behavior:** Click outside the popover div closes it (via `useRef` + `document.addEventListener('mousedown', ...)` — source: RESEARCH.md Pattern 4).

## Interaction Contract

### State Machine

```
Initial load → [MÊS] active → kpiMetrics filtered to current month

User clicks [SEMANA]   → [SEMANA] active, kpiMetrics filtered to current week
User clicks [MÊS]      → [MÊS] active, kpiMetrics filtered to current month
User clicks [ANTERIOR] → [ANTERIOR] active, kpiMetrics filtered to previous month
User clicks [PERSONALIZADO] → popover opens, [PERSONALIZADO] active (no filter applied yet)
  User selects start + end, clicks Aplicar → popover closes, custom filter applied
  User clicks outside popover → popover closes, previous filter state preserved
  User selects end < start → Aplicar button disabled, validation error shown

Custom range active: pill label changes from "PERSONALIZADO" to "dd/MM - dd/MM"
```

### Filter Persistence

Date filter state persists across Dashboard ↔ Negócios view switches (D-06). Follows same `useState` pattern as `category` in App.tsx — state lives at the App level.

### Stacking with State (UF) Filter

Date filter and state (UF) map click filter are independent and stack: both filters apply simultaneously. Changing the date filter does NOT reset `selectedState`. Source: RESEARCH.md Anti-patterns; CONTEXT.md Claude's Discretion.

### Map Behavior

The MapChart receives date-filtered rows (same as KPI cards). "Global" filter means the map choropleth also reflects the active date range. Source: RESEARCH.md Open Question 1 recommendation.

### DealsView

Date filter does NOT apply to the Negócios / DealsView table. DealsView always receives `data.rows` unfiltered. Source: RESEARCH.md Open Question 2 recommendation; no FILT requirement covers DealsView.
